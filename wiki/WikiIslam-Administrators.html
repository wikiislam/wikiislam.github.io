Vandalism Related

Lock down/protected Mode

This mode is enabled on the wiki to protect against ongoing vandalism which may involve multiple users, user accounts or IP addresses.

This mode disables anonymous edits and new user accounts.

Lock

(click and save page)

Unlock

(click and save page)









  • No Anonymous edits: (click on either link and save the page): On | Off

This disables anonymous edits from IP. Its helpful for spam or vandalism. The page WikiIslam:Forum can still be edited by IP users. For spam on that page, protect it temporarily separately.

Blocking/Unblocking

  • WikiIslam:Blocking Policy
  • Block user or IP | Block log | Edit Monitor log | old: Auto Block log
  • Range blocks: (for persistent vandalism using similar IP's) Range Blocks (mediawiki.org) | IP Range Block calculator
    • To ban IP address 111.x.x.x: 111.0.0.0/8
    • To ban IP address 111.222.x.x: 111.222.0.0/16
    • To ban IP address 111.222.333.x: 111.222.333.0/24

  • MediaWiki:Edit Monitor:
    • Edit Monitor log
    • Edit Monitor Info for Admins (clear ET Table now) - use after unblocking a user that has been blocked by AB by mistake.
    • System Messages related to Edit Monitor. For example: MediaWiki:em-block-message - template for message shown to users blocked by Edit Monitor.
    • Username lists (edit these lists)
      • em-bypass-1: Users who are new to the site and are still not considered completely trusted, should be on this list. Anyone can be added to this list without any risk.       • em-bypass-2: Users on this list are not monitored at all. Only users who are completely trusted should be on this list. Administrators do not have to be added to any lists.

Spam

IP addresses for spam can be blocked for 3-6 months:

  • Related template: Template:Captcha response - page that contains the anti-spam response
  • MediaWiki:Spam-blacklist - blacklist spam websites to prevent reinsertion of spam links

If a spam bot keeps recreating a certain page:

  • If you dont want to create the page see Wikipedia's help on protecting pages from being created. Steps for that should be:
    1. Delete the page.
    2. In the dropdown for that page, click "Protect"
      • You'll see protection options in a new "Create" box. Choose "Allow only autoconfirmed users".
      • Click "Confirm".
    3. After protection, you should see this in the Recent changes: Admin (Talk) protected "Spam page Title"‎ ‎[create=autoconfirmed] (indefinite)
  • Another way to protect a page from a spam bot is to write a hidden comment in the page, save and protect the page for a few months from IP and new users (if its predicted that genuine new and IP users will most likely not edit the page)

Use MediaWiki:Spam-blacklist to blacklist URL's from persistent spammers.

  • Clear Site Cache - The Site Caches store previously retrieved data and pages to speed up site performance. Clear the site cache after:
    • (a) changes to templates that are used on a lot of pages. Use it if pages or templates are not showing updates or their most recent versions.
    • (b) sidebar links (site menu on the left) are changed.
    • (c) a problem was detected on a mobile version of a page. The mobile cache will not refresh unless the cache is cleared.
    • If clearing the site cache does not solve the issue, the problem might be something else.
    • Clear the cache only occasionally or when necessary because the CPU usage increases slightly for some time after the cache is cleared.

Guidelines for Administrators

  • Page deletions: Page deletions should be done only after it is ascertained that no part of the page content can be re-used to better fit our guidelines
  • User blocks: User blocks should be done in accordance with the blocking policy
  • Reversal of admin actions (user blocks and deletions) should be done after consulting with the administrator who initiated the action
